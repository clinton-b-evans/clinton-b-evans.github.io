2. Control Flow and Loops

₪ Make Good Decisions ₪

With EQ fresh on the mind, this is the perfect time to explain control flows and loops. I am going to start with loops.

Have you ever had the same thought over and over... from like a time you did something in highschool, randomly pop into your brain at 3am? That's a loop. It usually continues until you get out of bed and kiss sleep good bye or you fall asleep.
For computer programming languages loops work a little like that. There are two key words when talking about loops:
1. For loops
2. While loops

For loops are when you have to do the same thing over and over for a set amount of times. Example:

I have 5 unpaid bills in the mail (or email).
My 1st of the month habit is like this:
1. Open bill
2. Cry
3. Pay bill
4. ->Recover

LUCKILY this will only happen to me 5 times. I know this because I only have 5 bills. I can bank of only having to go through this 5 times. Like the moon, loops have a darkside. If I left my code with numberOfBills = 5, then I would never pay anything more than 5 bills (sounds amazing). It's best practise to set the counter of the loop to the numberOfBills (or the length of the array)
That's where the dirty magic comes in. Our For loop will run for as mnany times as there are bills. This means if I only have 1 bill, the loop runs once. Amazing.

While loops are when you have a condition rather than a set number.
For example my dog has this while loop when he has just had a nightmare.
While human cannot be seen:
----- bark as loud as possible
----- wait 30 seconds

Those two things are repeated until I come through and cuddle him. A techie example could be:
Your online messenger could do something like that:
While "typing", show 3 dots movings

You ever said something stupid to someone and then those 3 bouncy dots just tourment you for hours (although it was only 2 minutes of wait time). There's another human loops:
While waiting for reply: (while reply = none)
---- check phone for reply
---- experience massive amounts of anxiety

This is the perfect time for a segway into:

Control Flows:

4. Arrays vs Objects

An Array is a list.
An Object is a thing with specific defined properties

let playlist = ['jams','doof doof music','rad jams','zero-cool']

This is an array. We can return the value of each item in the list by using it's index number. Just remember that computers start at 0 rather than 1. So to return 'jams' we do this
console.log(playlist[0])
Computers count from 0. So 0 is our 1st object.
console.log(playlist[3])
returns zero-cool.

Objects on the other hand are well objects. You define the object attributes (things it will have) like:
puppy = {
   name: 'Chester'
   age: '11.5 months'
}
And how old is puppy? We find out by using the key (name of property) like so:

console.log(puppy.name)
'Chester'
